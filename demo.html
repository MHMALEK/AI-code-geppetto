The idea

From Jira ticket to merged PR,
no human in the loop

1
POST a task
curl with title + description + Jira ID. Simulates a webhook from any source.
2
Agent searches the codebase
RAG index returns the exact functions, components, and hooks that matter for this task.
3
Agent writes + commits code
Reads files, makes precise edits, creates a branch, commits with a conventional message.
4
PR opened automatically
Pushes the branch and opens a pull request with a clear description of what changed.

The secret sauce

High-quality code indexing

Not chunks of 500 chars. AST-based semantic boundaries.

❌ NAIVE CHUNKING
...n error:\n  logger.error(
  `Failed to ${action}`
)\n  raise HTTPExce...

Broken mid-function. No context. Useless to the LLM.

✅ AST-BASED (tree-sitter)
// File: src/components/SupplierTable.tsx
// Type: component | Name: SupplierTable
// Imports: useQuery, Spinner, ...

const SupplierTable = ({ filters }) => {
  const { data, isLoading } = useQuery(...)
  return <Table ... />
}
🌳
tree-sitter
Parses TS/TSX/JS into AST. Extracts functions, hooks, components, classes at exact boundaries.
🔢
Rich embeddings
File path + type + name prepended before embedding. The model knows what it's reading.
🎯
Two-tier retrieval
Semantic search + exact symbol lookup. If task mentions SupplierTable, fetch it directly.

Quick start

Up and running in 3 steps

1. Configure

# fill in GCP project + OpenAI key
cp .env.template .env
gcloud auth application-default login

2. Start

./run.sh   # indexes repo on first run, then serves at :8000

3. Fire a task

curl -X POST http://localhost:8000/tasks \
  -H "Content-Type: application/json" \
  -d '{"jira_id":"DEV-1234",
      "title":"Add loading spinner",
      "description":"Show spinner while table loads"}'
